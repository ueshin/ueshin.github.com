Google Guice

Takuya UESHIN (2008年10月19日 14:26)
Siphonフレームワークのバックグラウンドで、Google GuiceというDIコンテナを使うことにしました。

DIコンテナといえば、日本ではSpring FrameworkやSeasar2が有名ですが、どちらもちょっとヘビー級になってしまっているので、DIxAOPに特化したGoogle Guiceを採用することにしました。

Siphonフレームワークを利用する際にGoogle Guiceのアノテーションが必要になる予定ですので、基本的な使い方を見てみます。

ちなみにGuiceは「じゅーす」と読みます。

Google Guiceの特徴

Google GuiceはDIコンテナの1つですが、次のような特徴があります。

  • DIxAOPコンテナに特化
  • XMLレス
  • 軽いらしい
  • スコープを自分で作成できる

調べ始めて間もないので余り確かな情報とはいえませんが。ｗ

とはいえ、スコープを自分で作成できるのは他のDIコンテナにはない特徴ではないでしょうか。

ダウンロード

ダウンロード、といっても本体はMaven2レポジトリにアップされていますのでpom.xmlファイルの設定だけで大丈夫です。

pom.xml

<project>

	・・・・

	<dependencies>
		<dependency>
			<groupId>com.google.code.guice</groupId>
			<artifactId>guice</artifactId>
			<version>1.0</version>
		</dependency>

	・・・・

	</dependencies>

	・・・・

</project>

使い方

Guice User's Guideのサンプルから、基本的な使い方を紹介します。

DIコンテナを使う場合には、インターフェイスとその実装クラスが必要になります。

Service.java

public interface Service {
  void go();
}

ServiceImpl.java

public class ServiceImpl implements Service {
  public void go() {
    ...
  }
}

ここで、Google Guiceはコンテナへの登録に、Moduleを使います。

MyModule.java

public class MyModule implements Module {
  public void configure(Binder binder) {
    binder.bind(Service.class)
      .to(ServiceImpl.class)
      .in(Scopes.SINGLETON);
  }
}

このServiceを利用するクライアント側のコードは次のようになります。

Client.java

public class Client {

  private final Service service;

  @Inject
  public Client(Service service) {
    this.service = service;
  }

  public void go() {
    service.go();
  }
}

このように、@Injectアノテーションでコンストラクタインジェクションを行うように指定しています。
このClientも、Google Guiceに登録してある必要があります。
[訂正] ClientクラスはGoogle Guiceに登録してある必要はないようです。

サンプルにはありませんでしたが、これらを実行するコードは以下のようになります。

Main.java

public class Main {
  public static void main(String[] args) {
    Injector injector = Guice.createInjector(new MyModule());
    Client client = injector.getInstance(Client.class);
    client.go();
  }
}
